## 3. 集成测试大纲

### IT-01: 端到端路由流程

**场景**: 用户发起请求 → 鉴权 → 路由 → 转发 → 响应

| 用例 | 流程 | 预期状态码 | 优先级 |
|------|------|:---:|:---:|
| auto 路由 - 办公 | `model:"auto"`, 办公内容, 权限内 | 200 + 流式响应 | P0 |
| auto 路由 - 代码 | `model:"auto"`, 代码内容, 权限内 | 200 + 流式响应 | P0 |
| 指定模型 | `model:"gpt-4o"`, 在权限内 | 200 | P0 |
| 指定模型 - 越权 | `model:"gpt-4o"`, 不在权限内 | 403 + 提示可用模型 | P0 |
| 无 Key | 无 Auth Header | 401 | P0 |
| 无效 Key | `Bearer sk-invalid` | 401 | P0 |
| 已吊销 Key | 吊销后的 Key | 401 | P0 |
| 用户禁用 | 禁用用户 | 403 | P0 |
| 熔断用户 | 100% 用量 | 429 | P0 |
| 流式响应透传 | `stream:true` | SSE 格式正确 | P1 |
| 全部渠道不可用 | 所有渠道禁用 | 502 | P1 |

### IT-02: 熔断恢复流程

| 用例 | 流程 | 预期 | 优先级 |
|------|------|------|:---:|
| 渐进式告警 | 50%→80%→90%→100% | 依次触发 level 1,2,3 | P0 |
| 熔断后拒绝 | 100% 后发请求 | 429 | P0 |
| 管理员恢复 | 提升额度 + 解除熔断 | 请求恢复 200 | P0 |
| 跨月自动重置 | 月初自动重置计数 | 新周期可用 | P1 |

### IT-03: 渠道健康检查

| 用例 | 流程 | 预期 | 优先级 |
|------|------|------|:---:|
| 健康渠道正常 | 定时检查通过 | healthy | P0 |
| 故障自动禁用 | 3次失败 | 禁用 + 通知 | P0 |
| 故障后降级 | 首选故障 → 自动选备选 | 返回 200 | P1 |
| 恢复自动启用 | 故障恢复后 | 自动启用 | P1 |

### IT-04: 包月模型限制

| 用例 | 流程 | 预期 | 优先级 |
|------|------|------|:---:|
| 次数未达上限 | 已调用 50/10000 | 正常返回 | P0 |
| 次数达上限 | 已调用 10000/10000 | 自动禁用 + 通知 | P0 |
| 不限次 | call_limit=0 | 永不限制 | P1 |

## 5. 性能测试

### PT-01: 网关延迟

| 指标 | 目标 | 测试方法 |
|------|------|------|
| 鉴权延迟 | ≤ 5ms | Redis 命中, 简单请求 |
| 路由决策延迟 | ≤ 10ms | 内容分类 + 渠道选择 |
| 总额外延迟 | ≤ 50ms (P99) | 端到端 proxied 请求 |
